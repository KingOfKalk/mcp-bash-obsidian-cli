# PARA Method — Compact Guide & Cheat Sheet

**Author:** Tiago Forte | **Books:** "Building a Second Brain" (2022), "The PARA Method" (2023)
**Core idea:** Organize all digital information into exactly 4 categories, sorted by actionability — not by topic.

---

## 1. The Four Categories

### P — Projects
- Series of tasks linked to a goal, **with a deadline**
- Has a clear finish line → can be marked "done"
- Uses completion verbs: *finalize, ship, deliver, publish, launch*
- Keep 10–15 active max
- Examples: "Launch website by March 15", "Write Q1 report", "Renovate bathroom"

### A — Areas of Responsibility
- Sphere of activity with a **standard to maintain over time**
- No end date, never "done" — requires ongoing attention
- Uses maintenance verbs: *manage, ensure, oversee, maintain*
- Examples: Health, Finances, Team Management, Kids, Home

### R — Resources
- Topic or theme of **ongoing interest** (not responsibility)
- Reference library for things you're curious about
- Distinction from Areas: interest ≠ responsibility
- Examples: Photography, Machine Learning, Coffee, Stoicism

### A — Archives
- Inactive items from the other three categories
- Completed projects, dropped interests, paused areas
- Never delete — just archive. Remains searchable.

---

## 2. Quick Classification (Decision Tree)

Ask in this order, stop at the first "yes":

1. **Does it belong to an active project with a deadline?** → Projects
2. **Does it relate to an area I'm responsible for?** → Areas
3. **Is it a topic I'm interested in?** → Resources
4. **None of the above / no longer active?** → Archives

---

## 3. Projects vs. Areas — The Critical Distinction

This is the #1 source of confusion and the #1 cause of broken systems.

| Test         | Project                    | Area                     |
| ------------ | -------------------------- | ------------------------ |
| Finish line? | Yes — clear end state      | No — ongoing forever     |
| Deadline?    | Yes                        | No                       |
| Verb type    | Complete, deliver, ship    | Manage, maintain, ensure |
| Example      | "Run a marathon"           | "Health"                 |
| Example      | "Hire senior dev by April" | "Team Management"        |

**Rule of thumb:**
- A goal without a project = a dream
- A project without a goal = a hobby

---

## 4. Folder Structure

```
0-Inbox/          ← capture buffer, process weekly
1-Projects/       ← active, time-bound work
2-Areas/          ← ongoing responsibilities
3-Resources/      ← reference & interests
4-Archives/       ← inactive items from 1–3
```

- Number prefixes force correct sort order
- Mirror this structure identically across all tools (notes, files, tasks)
- Never create empty folders — only add structure when you have content
- Max depth: 4 levels

---

## 5. PARA in the Bigger Picture: CODE

PARA is step 2 ("Organize") of Forte's CODE method:

- **C — Capture:** Save what resonates. Don't over-filter.
- **O — Organize:** Sort into PARA by actionability.
- **D — Distill:** Refine notes via Progressive Summarization (bold → highlight → summarize → remix). Most notes stay at Layer 1–2.
- **E — Express:** Turn knowledge into output using Intermediate Packets (reusable chunks: outlines, drafts, summaries).

**PARA + GTD:**
- GTD manages verbs (actions, next steps)
- PARA manages nouns (notes, files, reference material)
- Both project lists should mirror each other

---

## 6. Maintenance Rules

- **Weekly Review (15–30 min):** Clear inbox, scan calendar ±2 weeks, process notes into PARA, set priorities
- **Projects:** Review weekly
- **Areas:** Review monthly/quarterly
- **Resources:** Review only when you act on a topic
- **Archives:** No review needed — just searchable storage
- **Archive aggressively** — if it's not active, move it
- **Information bankruptcy:** If overwhelmed, archive everything, start fresh. What matters will resurface.

---

## 7. Common Mistakes

- Treating Areas as Projects (no deadline = not a project)
- Over-engineering with tags, colors, nested hierarchies before filing a single note
- Hoarding inactive items in active folders
- Confusing interests with commitments
- Trying to migrate everything at once instead of starting lean
- Inconsistent naming across tools

---

## 8. Agent Instructions

When classifying information for a PARA system:

1. Run the 4-question decision tree (Section 2) in strict order
2. Apply the verb test (Section 3) to distinguish Projects from Areas
3. Enforce the 10–15 active projects ceiling
4. Default to Archives when actionability is uncertain
5. Never create speculative folders — only file what exists
6. When reorganizing: archive first, rebuild from what's needed
